LoRa 패킷 로거
====================

1. 소개
----------------

이 소프트웨어는 JSON 구성을 사용하여 LoRa 집중 장치를 설정하는 데 사용됩니다.
파일을 생성한 다음 로그 파일에 수신된 모든 패킷을 무기한 기록합니다.
사용자가 응용 프로그램을 중지합니다.
필터링이 수행되지 않고 올바른 RF가 있는 LoRa 패킷인 모든 패킷
매개변수(주파수, 데이터 전송률, 대역폭)가 로그에 나타나야 합니다.

2. 종속성
----------------

이 프로그램은 다음과 같이 Parson 라이브러리(http://kgabis.github.com/parson/)를 사용합니다.
JSON 구문 분석을 위한 Krzysztof Gabis.
매우 실용적이고 잘 작성된 라이브러리에 대해 그에게 많은 감사를 드립니다.

이 프로그램은 수신을 위한 LoRa 집중기 HAL 사용의 전형적인 예입니다.
패킷.

높은 수준의 함수(loragw_hal에 포함된 함수)만 사용되므로
HAL이 적절한 버전과 일치한다고 가정하면 하드웨어 종속성이 없습니다.
하드웨어의.
수신된 패킷의 데이터 구조는 이름으로 액세스됩니다(즉,
바이너리 수준)에 영향을 미치지 않고 API에 새로운 기능을 추가할 수 있습니다.
전혀 그 프로그램.

libloragw 라이브러리의 v1.3.0으로 테스트되었으며 호환되어야 합니다.
API가 하향 호환된다고 가정하면 라이브러리의 모든 이후 버전과 함께 사용합니다.

3. 사용법
----------

응용 프로그램을 중지하려면 Ctrl+C를 누릅니다.

응용 프로그램을 시작할 때 유일한 선택적 매개 변수는 로그 회전입니다.
시간(초).

프로그램이 구성 파일을 고려하는 방식은 다음과 같습니다.
 * debug_conf.json이 있는 경우 이를 구문 분석하면 나머지는 무시됩니다.
 * global_conf.json이 있으면 구문 분석하고 다음 파일을 찾습니다.
 * local_conf.json이 있으면 구문 분석
일부 매개변수가 전역 및 로컬 구성 파일 모두에 정의된 경우
지역 정의가 전역 정의를 덮어씁니다.

전역 구성 파일은 전체에서 정확히 동일해야 합니다.
네트워크, 모든 전역 매개변수("센서" 무선 채널에 대한 매개변수) 포함
그리고 바람직하게는 각각에 특정한 매개변수에 대한 기본 "안전한" 값
게이트웨이(예: 기본 MAC 주소 지정).

특정 무선 대역(예: ETSI
868MHz 대역) Makefile에 의해 즉시 사용 가능한 global_conf.json 파일이 생성됩니다.
'LoRa MAC' 네트워크 애플리케이션에 일반적으로 사용되는 채널 집합입니다.
무선 대역을 지정하지 않으면 빈 global_conf.json이 생성되고
필요한 설정으로 채워야 합니다.

로컬 구성 파일에는 각각에 특정한 매개변수가 포함되어야 합니다.
게이트웨이(예: MAC 주소, 백홀 무선 채널의 주파수).

각 구성 파일에서 프로그램은 다음과 같은 JSON 개체를 찾습니다.
LoRa 집중기 보드에 대한 매개변수를 포함해야 하는 "SX1301_conf"
(RF 채널 정의, 모뎀 매개변수 등) 및 다른 JSON 객체
게이트웨이 매개변수(게이트웨이 MAC 주소,
LoRa MAC 컨트롤러의 IP 주소, 네트워크 인증 매개변수 등).

JSON 구성 형식에 대해 자세히 알아보려면 제공된 JSON 파일을 읽어보세요.
API 문서. 전용 문서는 나중에 사용할 수 있습니다.

수신된 패킷은 이름에 MAC 주소가 포함된 CSV 파일에 저장됩니다.
16진수 형식의 게이트웨이 및 로그 시작 시간의 UTC 타임스탬프
ISO 8601 권장 컴팩트 형식:
yyyymmddThhmmssZ(예: 2013년 10월 9일 오후 5:23:45 UTC의 경우 20131009T172345Z)

지속적인 모니터링을 위해 현재 닫혀 있는 로그 파일을 닫고
매 시간마다 새 항목이 열립니다(기본적으로 회전 간격은
-r 명령줄 옵션을 사용하는 사용자).
로그 파일이 회전하는 동안 패킷이 손실되지 않습니다.
현재 로그 파일을 제외한 모든 로그 파일을 수정, 업로드 및/또는 삭제할 수 있습니다.
프로그램 실행에 대한 결과 없이.
